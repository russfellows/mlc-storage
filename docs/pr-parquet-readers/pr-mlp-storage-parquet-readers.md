### `tests/unit/test_parquet_reader.py` (new, +630 lines) ✅ 59/59 passing

Comprehensive unit-test suite for all parquet reader components. No live S3 endpoint or minio server required — all storage calls are backed by in-process mocks over real in-memory parquet bytes (3 row groups × 8 rows, generated by pyarrow).

**Test classes:**

| Class | Tests | What it covers |
|---|---|---|
| `TestFormatTypeParquet` | 3 | `FormatType.PARQUET` value, `get_enum()` round-trip, other types unaffected |
| `TestS3RangeFile` | 17 | `seek`/`tell`/`read` for SEEK_SET/SEEK_CUR/SEEK_END, read clamping at EOF, size lazy-fetch, `closed` property, live pyarrow `ParquetFile` integration, row-group data correctness |
| `TestMinioRangeFile` | 11 | Same interface via mocked minio client, `stat_object` call verification, live pyarrow integration |
| `TestParquetReaderS3Iterable` | 26 | `open()` tuple/offset correctness, `get_sample()` for first/middle/last row group, cache hit (same object, no re-fetch), all 24 samples routed to correct row group, LRU eviction bounded by `row_group_cache_size`, eviction order, `close()` evicts only the closed file's entries, capability flags, URI absolute/relative/leading-slash handling, column projection |
| `TestReaderFactoryParquetRouting` | 2 | Factory import verification, source-level check that `FormatType.PARQUET` branch exists |

Two production bugs were caught and fixed during test authoring:
1. `_S3RangeFile` and `_MinioRangeFile` were missing `closed` property and `close()` — required by pyarrow 23+
2. `_args` must be set before `__init__` is called in subclasses — test exposed this initialization ordering constraint
